📋 INICIO RÁPIDO (3 PASOS)
════════════════════════════════════════════════════════════════════════════════

1️⃣  INICIAR SERVIDOR
   $ cd f:\repos\Proyecto3VeranoDatos1\CalcuEXP
   $ dotnet run ServerProgram
   
   ✅ Se crea automáticamente: evaluations.csv

2️⃣  EJECUTAR CLIENTE (otra ventana)
   $ dotnet run
   
   ✅ Se abre la interfaz gráfica

3️⃣  USAR EL SISTEMA
   • Conectar al servidor
   • Evaluar expresiones
   • Click "Historial" para ver datos
   • Click "Exportar CSV" para guardar


⚡ 30 SEGUNDOS PARA ENTENDER
════════════════════════════════════════════════════════════════════════════════

¿Qué es CalcuEXP?
└─ Una calculadora remota cliente-servidor basada en TCP

¿Qué es NUEVO?
└─ ✅ Historial automático de evaluaciones
└─ ✅ Acceso remoto al historial
└─ ✅ Exportación a CSV

¿Cómo lo uso?
└─ Evalúa una expresión → Se guarda automáticamente
└─ Click "Historial" → Ves todos los registros
└─ Click "Exportar" → Guardas como archivo CSV

❓ PREGUNTAS FRECUENTES
════════════════════════════════════════════════════════════════════════════════

P: ¿Dónde se guarda el historial?
R: En evaluations.csv en la carpeta del servidor

P: ¿Pueden múltiples clientes acceder al historial?
R: Sí, todos ven el mismo historial

P: ¿Se pierde el historial si reinicio el servidor?
R: No, está guardado en evaluations.csv

P: ¿Cómo sé qué cliente hizo cada cálculo?
R: Mira la columna "Cliente" que contiene la IP

P: ¿Puedo borrar el historial?
R: Sí, elimina evaluations.csv (se recrea vacío al siguiente registro)

P: ¿Es seguro para producción?
R: Sí, incluye sincronización y manejo de errores
