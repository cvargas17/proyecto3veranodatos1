📊 ESTRUCTURA DE DATOS - ARCHIVO CSV
═══════════════════════════════════════════════════════════════════════════════

Fecha,Hora,Expresión,Resultado,Cliente
2025-01-16,14:30:45,"2+2","4",127.0.0.1
2025-01-16,14:31:12,"10*5","50",127.0.0.1
2025-01-16,14:32:00,"100/2","50",192.168.1.100

🔑 Campos:
├─ Fecha       : Día de evaluación (YYYY-MM-DD)
├─ Hora        : Hora exacta (HH:MM:SS)
├─ Expresión   : La expresión matemática/lógica
├─ Resultado   : El resultado o ERROR:descripción
└─ Cliente     : IP del cliente que solicitó

✨ CARACTERÍSTICAS PRINCIPALES
═══════════════════════════════════════════════════════════════════════════════

✅ Guardado Automático
   • Cada evaluación se guarda en evaluations.csv
   • Sin intervención del usuario

✅ Información Completa
   • Fecha y hora exacta de cada cálculo
   • Expresión original
   • Resultado obtenido
   • IP del cliente que lo realizó

✅ Acceso Remoto
   • Los clientes pueden descargar el historial del servidor
   • Sin necesidad de acceso físico al servidor

✅ Exportación de Datos
   • Los usuarios pueden guardar el historial como CSV
   • Compatible con Excel, LibreOffice, etc.

✅ Seguridad Thread-Safe
   • Sincronización correcta para múltiples clientes
   • Protección del archivo CSV contra accesos simultáneos

✅ Escalabilidad
   • Buffers aumentados para datos grandes
   • Soporta múltiples clientes concurrentes
